Ticket 1.1: Interface Extraction & Domain Encapsulation
Goal: Define the standard contract that the main game loop will use to interact with any loaded song, and encapsulate the existing MIDI logic into a concrete implementation of that contract.

Step 1: Define the Interface
Create a new file IPlayableSong.cs in a Core/Interfaces folder. This interface must hide all underlying data structures (whether MIDI or PSARC) from the main game loop.

C#
using ImGuiNET;
using System;

public interface IPlayableSong : IDisposable {
    string Title { get; }
    string Artist { get; }
    TimeSpan TotalDuration { get; }
    
    // Called once per frame to update internal state/positions
    // audioTimeMs is the exact current playback position of the audio engine
    void Update(float audioTimeMs); 
    
    // Called once per frame during the ImGui render phase
    void Draw(ImDrawListPtr drawList); 
}

The Draw Method: Move Openthesia's ImGui.GetWindowDrawList().AddRectFilled(...) logic (the falling blocks and the 88-key piano keyboard at the bottom of the screen) into this method.

public void RenderFrame(float currentAudioTimeMs) {
    ImGui.Begin("Main Highway Window");
    
    if (_currentSong != null) {
        ImDrawListPtr drawList = ImGui.GetWindowDrawList();
        
        // 1. Update internal state based on audio time
        _currentSong.Update(currentAudioTimeMs);
        
        // 2. Draw the specific media format to the screen
        _currentSong.Draw(drawList);
    } else {
        ImGui.Text("Drop a .mid file to begin.");
    }
    
    ImGui.End();
}
