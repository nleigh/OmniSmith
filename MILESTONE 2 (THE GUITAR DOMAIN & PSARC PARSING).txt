MILESTONE 2 (THE GUITAR DOMAIN & PSARC PARSING)
Context for LLM: You are the Lead C# Engineer on the "OmniRhythm" engine. We have successfully completed Milestone 1 (Core Engine Refactoring) and have our IPlayableSong interface.

Objective: We are now building Domain B: The Guitar/CDLC Domain. You will implement the data structures to hold guitar-specific notation, use Rocksmith2014.NET to parse .psarc files, and use vgmstream-cli to decode the proprietary audio.

Please review the detailed requirements for Tickets 2.1, 2.2, and 2.3 below. Acknowledge these requirements, and then write the C# code to implement them.

Ticket 2.1: Guitar Data Structures
Goal: Define the domain-specific models that hold Rocksmith data, isolated entirely from the Piano domain.

Step 1: The Techniques Enum
Create Domains/Guitar/Models/NoteTechnique.cs. Use a bitmask (Flags) because a single note can have multiple techniques (e.g., a Palm Muted Slide).

C#
using System;

[Flags]
public enum NoteTechnique {
    None = 0,
    HammerOn = 1 << 0,
    PullOff = 1 << 1,
    Slide = 1 << 2,
    Bend = 1 << 3,
    PalmMute = 1 << 4,
    Harmonic = 1 << 5,
    PinchHarmonic = 1 << 6,
    Vibrato = 1 << 7,
    Sustain = 1 << 8
}
Step 2: Note and Chord Models
Create Domains/Guitar/Models/GuitarNote.cs and GuitarChord.cs.

GuitarNote: Must contain Time (ms), String (0-5), Fret (0-24), Duration (ms). Add fields for expressive data: Techniques (enum), SlideToFret, and BendValue.

GuitarChord: Must contain Time (ms), a generic Name string (e.g., "Am"), and a List<GuitarNote> containing the individual strings played in that chord.

Step 3: The GuitarSong Implementation
Create Domains/Guitar/GuitarSong.cs that implements IPlayableSong from Milestone 1.

It should contain List<GuitarNote> Notes and List<GuitarChord> Chords.

It needs an AudioFilePath string property (to store the path to the decoded .wav file).

For now, leave the Update() and Draw() methods blank (we will implement the ImGui highway rendering in Milestone 3).

Ticket 2.2: The Parser Service
Goal: Extract the XML data from the PSARC and map it to our custom GuitarNote models.

Step 1: Rocksmith2014.NET Integration
We assume the project references the Rocksmith2014.PSARC and Rocksmith2014.XML packages.

Step 2: The Parser Class
Create Domains/Guitar/Services/RocksmithParser.cs. Create an async method ParsePsarcAsync(string filePath, string preferredArrangement = "lead").

Step 3: PSARC Extraction & XML Mapping Logic
Implement the following logic flow:

Open Archive: Open the PSARC file.

Find Arrangement: Search the PSARC entries for an XML file matching the preferredArrangement (e.g., ends with _lead.xml).

Deserialize: Use Rocksmith2014.XML.InstrumentalArrangement.Read(stream) to parse the XML.

Handle Difficulty Levels: Rocksmith stores notes in Levels. For CDLC, the full chart is almost always in the highest difficulty level. Find the Level with the highest Difficulty integer.

Map Notes: Iterate through level.Notes. Map them to our GuitarNote object. Multiply Rocksmith's native time values (which are usually in seconds) by 1000 to convert to Milliseconds for our engine.

Map Chords: Iterate through level.Chords. Map them to our GuitarChord object. (Note: Rocksmith XML often defines chord templates separately. You may need to look up the chord template ID to get the fret/string data).

Ticket 2.3: Audio Extraction (The WEM Pipeline)
Goal: Rocksmith's audio is trapped in proprietary .wem files. We must extract it and decode it so the core audio engine can play it.

Step 1: Extract WEM from PSARC
Inside RocksmithParser.cs, after mapping the XML, locate the main audio entry in the PSARC (usually ends with _wem). Extract this stream and write it to disk in the OS Temp directory (Path.GetTempPath()), naming it something like temp_audio.wem.

Step 2: The vgmstream Wrapper
Create a private helper method Task<string> DecodeWemToWavAsync(string wemFilePath).
Use System.Diagnostics.Process to execute vgmstream-cli.

Command Line Arguments: -o "[output_path.wav]" "[input_path.wem]"

Ensure CreateNoWindow = true and UseShellExecute = false so no command prompt flashes on the user's screen.

Step 3: Finalize GuitarSong
Once the process exits successfully, delete the temp_audio.wem file to save disk space. Set the resulting .wav file path to the GuitarSong.AudioFilePath property, and return the fully populated GuitarSong object back to the main loop.

Definition of Done: You can pass a valid .psarc file path into RocksmithParser.ParsePsarcAsync(). It returns a GuitarSong object containing a populated list of notes/chords and a valid file path to a playable .wav file in the Temp directory.
