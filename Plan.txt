OMNIRHYTHM: TECHNICAL SPECIFICATION
1. Architectural Strategy: The Universal Timeline
Do not build separate players for MIDI and CDLC. Instead, we will abstract the parsed data into a Universal Timeline.

Input Layer: Specialized parsers read the raw files (MIDI parser for .mid, Rocksmith2014.NET for .psarc/SNG).

Abstraction Layer: Parsers convert their proprietary note data into a generic OmniNote data structure.

Render Layer: The engine reads the OmniNote array. If the context is Piano, it draws falling bars to 88 keys. If the context is Guitar, it draws 3D notes moving down a 6-string highway.

2. Tech Stack & Dependencies
Core: C# / .NET (Matching Openthesia's current version).

CDLC Parsing: Rocksmith2014.NET (pull via NuGet or source). This will handle PSARC decryption, extraction, and XML/SNG parsing.

Audio Engine: ManagedBass (BASS audio library wrapper). Essential for handling real-time audio playback, volume control, and high-quality time-stretching/pitch-shifting without altering the original file.

Audio Decoding (The WEM Pipeline): vgmstream-cli and ffmpeg.exe. These must be bundled as external binaries. The C# app will use System.Diagnostics.Process to transparently convert Rocksmith's proprietary .wem audio files into .ogg or .wav in a system Temp directory prior to playback.


OMNIRHYTHM: MODULAR TECH SPEC
1. Architectural Strategy: Separation of Domains
Instead of forcing everything into one "Omni" class, we will split the app into three distinct layers:

The Shared Core Engine: Manages global state, audio playback, time synchronization, and the ImGui UI.

Domain A (Piano/Openthesia Legacy): Contains the MIDI parser (DryWetMIDI) and the 88-key falling-block renderer.

Domain B (Guitar/Slopsmith Port): Contains the PSARC parser (Rocksmith2014.NET) and the 6-string 3D highway renderer.

2. Core Abstractions (Interfaces)
To allow the Core Engine to handle both domains without knowing their specific details, we will define strict interfaces.

C#
// Shared by both domains
public interface IPlayableSong {
    string Title { get; }
    string Artist { get; }
    TimeSpan TotalDuration { get; }
    void Update(float currentAudioTimeMs);
    void Draw(ImDrawListPtr drawList); // ImGui.NET drawing context
}

// Domain A: Piano Implementation
public class PianoSong : IPlayableSong {
    public List<PianoNote> Notes { get; set; }
    public void Draw(ImDrawListPtr drawList) {
        // Openthesia's existing falling block drawing logic goes here
    }
}

// Domain B: Guitar Implementation
public class GuitarSong : IPlayableSong {
    public List<GuitarNote> Notes { get; set; }
    public List<GuitarChord> Chords { get; set; }
    public void Draw(ImDrawListPtr drawList) {
        // Slopsmith's 3D highway drawing logic goes here
    }
}
