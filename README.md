# OmniSmith

Combination of Slopsmith and OmniSmith to create an app to play songs.
- **My fork of OmniSmith:** https://github.com/nleigh/OmniSmith
- **Omnislop Reference:** https://github.com/byrongamatos/slopsmith

Just a play project testing capabilities of agent tooling (Hosted + local).
Spec generated by Gemini, implemented by local 5090 with gamma/qwen

## 🚀 Build & Run Instructions

To build and run the application locally, you must have the **.NET 6.0 SDK** installed.

**Quick Run (Debugging):**
```bash
dotnet run --project Openthesia/Openthesia.csproj
```

**Build Release Binary:**
```bash
dotnet publish Openthesia/Openthesia.csproj -c Release --self-contained true -p:PublishSingleFile=true
```

## 🏗️ Project Design & Plan

The system is transitioning to a **Domain-based Strategy Pattern** (using `IPlayableSong`):
- **Core Engine:** Manages global state, application loop, and handles window rendering independently of the music file type. Includes an embedded **SQLite Database** (`Microsoft.Data.Sqlite`) to seamlessly index and paginate thousands of files.
- **Piano Domain:** Evaluates traditional `.mid` files via DryWetMIDI, rendering falling 2D blocks and interacting with plugins.
- **Guitar Domain:** Leverages `Rocksmith2014` libraries and `vgmstream-cli` to parse Rocksmith 2014 CDLC schemas natively in C#. Provides a 3D perspective-projected GUI mirroring standard rhythm games mapping strings to screen depth.
- **Web Plugins:** Natively integrated C# service hooks allowing in-app browsing and downloading of tablatures or CDLC architectures (ported from Slopsmith).

All local agent development tasks and step-by-step blueprints are documented in the `Tickets/` directory.

## 🔄 Agent instructions for Slopsmith Sync

OmniRhythm actively ports logic and UI styling from the upstream web-based Slopsmith repository (`byrongamatos/slopsmith`). To bring in new features:

1. **Review Upstream Commits:** Periodically check `https://github.com/byrongamatos/slopsmith/commits/main/` or use Git tools to pull the remote log.
2. **Identify Core Changes:** Look for modifications in Slopsmith's `lib/` (parsing/audio pipelines), `static/` (rendering/UI logic like `highway.js`), or `plugins/` (scrapers and services).
3. **C# Translation:** Do not directly port JavaScript/Python syntaxes. Instead, interpret the mathematical, structural, or logical changes made upstream and implement them idiomatically in the C# `OmniRhythm` structure (e.g. updating ImGui components, `GuitarSong.Draw()`, or `RocksmithParser`).
4. **Update Sync Tracker:** Whenever an upstream commit is fully ported, update the local target metadata (e.g., `slopsmith_sync_state.json`) with the latest commit SHA to keep track of parity.
